- 리소스 정리 체크리스트 예시
  - EC2 인스턴스 Terminated 확인
  - EBS 볼륨(미사용 포함) 삭제 확인
  - Elastic IP Release 확인(할당했다면)
  - Internet Gateway Detach 및 삭제 확인
  - VPC 및 Subnet/Route Table 삭제 확인
  - (해당 시) NAT Gateway 삭제 확인
  - (해당 시) ELB/ALB 삭제 확인
  - (해당 시) RDS 삭제 확인
  - (권장) 모든 리소스 삭제 후 Billing Dashboard에서 과금 항목이 남지 않는지 확인

AWS는 "안에 들어있는 내용물(서버)을 먼저 비워야, 겉에 있는 껍데기(네트워크, 방화벽)를 부술 수 있다"는 규칙이 있습니다. 따라서 **반드시 아래에 적힌 순서대로 진행**하셔야 오류(종속성 에러)가 나지 않습니다.

---

### 🗑️ 1단계: 요금 발생의 핵심 원인 제거 (EC2 & 보안 그룹)

가장 먼저 돈이 나갈 수 있는 컴퓨터 본체를 부수고, 그 앞을 지키는 경비원(방화벽)을 해고하는 단계입니다.

- [ ] **1. EC2 인스턴스 종료 (가장 중요)**
  - [EC2] ➔ [인스턴스] 메뉴로 이동
  - `mission-web-server` 체크 ➔ 우측 상단 [인스턴스 상태] ➔ **[인스턴스 종료(Terminate)]** 클릭
  - ⏳ *주의: 상태가 '종료 중(Shutting down)'을 거쳐 완전히 **'종료됨(Terminated)'**으로 바뀔 때까지 1~2분 정도 충분히 기다려야 다음 단계를 진행할 수 있습니다.*
- [ ] **2. 보안 그룹 삭제**
  - [EC2] ➔ 왼쪽 메뉴 [보안 그룹]으로 이동
  - `mission-web-sg` 체크 ➔ 우측 상단 [작업] ➔ **[보안 그룹 삭제]** 클릭
- [ ] **3. AWS 키 페어 삭제**
  - [EC2] ➔ 왼쪽 메뉴 [키 페어]로 이동
  - `mission-key` 체크 ➔ 우측 상단 [작업] ➔ **[삭제]** 클릭

---

### 🌍 2단계: 네트워크 영토 반납 (VPC)

서버가 사라졌으니, 이제 서버가 있던 땅과 대문, 길을 한 번에 반납할 차례입니다.

- [ ] **4. VPC 한 번에 삭제**
  - 화면 상단 검색창에 `VPC` 검색하여 이동
  - 왼쪽 메뉴 [내 VPC] ➔ `mission-vpc` 체크
  - 우측 상단 [작업] ➔ **[VPC 삭제]** 클릭
  - 확인 창에 `delete`를 타이핑하고 삭제 클릭
  - *💡 꿀팁: VPC를 삭제하면 그 안에 속해 있던 서브넷(`mission-public-subnet`), 인터넷 게이트웨이(`mission-igw`), 라우팅 테이블(`mission-public-rt`)이 마법처럼 한 번에 묶여서 자동으로 삭제됩니다.*

---

### 🔒 3단계: 권한 및 계정 삭제 (IAM)

안전을 위해 만들었던 부캐(IAM 사용자)와 출입증(정책)을 폐기하여 계정 탈취 위험을 0%로 만듭니다. (이 작업은 루트 계정이나 IAM 관리 권한이 있는 계정으로 로그인해서 하시는 것이 좋습니다.)

- [ ] **5. IAM 사용자 삭제**
  - 상단 검색창에 `IAM` 검색하여 이동
  - 왼쪽 메뉴 [사용자] ➔ `mission-user` 클릭 ➔ 우측 상단 **[삭제]** 클릭
- [ ] **6. IAM 정책 삭제**
  - 왼쪽 메뉴 [정책] ➔ `Mission-EC2-VPC-Policy` 검색 및 체크 ➔ 우측 상단 [작업] ➔ **[삭제]** 클릭

---

### 💻 4단계: 내 PC 및 외부 사이트 흔적 지우기 (개인정보 관리)

AWS 밖(내 맥북과 No-IP)에 남아있는 찌꺼기를 치우는 마지막 단계입니다.

- [ ] **7. 로컬 맥북에 있는 `.pem` 열쇠 파일 지우기**
  - 맥북 터미널을 열고 키 파일이 있는 폴더(예: Downloads)로 이동
  - `rm mission-key.pem` 명령어를 입력하여 로컬 컴퓨터에 남은 마스터 키를 영구 삭제
- [ ] **8. No-IP 도메인 삭제**
  - No-IP 웹사이트 로그인 ➔ Dashboard 이동
  - [Dynamic DNS] ➔ [No-IP Hostnames] 메뉴 클릭
  - 시연에 사용했던 도메인(`mission61.ddns.net`) 우측의 **휴지통(Delete) 아이콘** 또는 **[Modify] ➔ [Delete]**를 클릭하여 IP 매핑 정보 삭제
