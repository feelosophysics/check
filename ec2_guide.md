# 🚀 왕초보를 위한 AWS EC2 완벽 정복 가이드 (결제/가입 이후)

환영합니다! 무사히 무료 플랜(Free Tier) 가입까지 마치셨군요. 처음 AWS 콘솔(웹 화면)에 접속하면 메뉴가 너무 많아서 "이게 다 뭐야..." 싶으실 겁니다. 정상입니다!

이 가이드는 **"AWS를 처음 써보는 사람"**의 눈높이에 맞춰, **클릭해야 할 버튼의 위치**까지 아주아주 상세하게 설명합니다. 우리가 만든 결정(한국어 환경, 수동 네트워크 구성, 우분투, 도커, 커스텀 IAM 정책)에 맞춰 단계별로 천천히 따라와 주세요.

---

## 🔒 1단계: 루트 계정 대신, 안전한 "IAM 사용자" 만들기

AWS에서는 가입할 때 사용한 이메일 계정을 **루트 계정(Root Account)**이라고 부릅니다. 이 계정은 모든 권한을 가진 '신'과 같아서, 해킹당하면 요금 폭탄을 맞을 수 있습니다. 따라서 미션의 요구사항대로 루트 계정을 쓰지 않고, **이번 실습에 딱 필요한 권한만 가진 부캐(IAM 사용자)**를 만들겠습니다.

1. **IAM 서비스 들어가기**
   - 로그인 직후 화면 상단 중앙에 있는 **검색창**을 클릭합니다.
   - 영어로 `IAM` 이라고 입력합니다.
   - 아래 검색 결과에서 **[IAM]** (AWS 리소스에 대한 액세스 관리)를 클릭합니다.

2. **커스텀 정책(Policy) 만들기**
   - 왼쪽 메뉴에서 **[정책(Policies)]**을 클릭합니다.
   - 오른쪽 화면 상단의 주황색 **[정책 생성]** 버튼을 클릭합니다.
   - 정책 편집기 화면에서 **[JSON]** 탭을 클릭합니다.
   - 원래 적혀있던 코드를 모두 지우고, 아래의 코드를 복사해서 붙여넣습니다. (이 코드는 이번 미션에 필요한 VPC, EC2 관련 기능만 허용하는 완벽한 최소 권한 코드입니다.)

```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": [
                "ec2:Describe*",
                "ec2:CreateVpc",
                "ec2:DeleteVpc",
                "ec2:ModifyVpcAttribute",
                "ec2:CreateSubnet",
                "ec2:DeleteSubnet",
                "ec2:ModifySubnetAttribute",
                "ec2:CreateInternetGateway",
                "ec2:AttachInternetGateway",
                "ec2:DetachInternetGateway",
                "ec2:DeleteInternetGateway",
                "ec2:CreateRouteTable",
                "ec2:DeleteRouteTable",
                "ec2:CreateRoute",
                "ec2:DeleteRoute",
                "ec2:AssociateRouteTable",
                "ec2:DisassociateRouteTable",
                "ec2:CreateSecurityGroup",
                "ec2:DeleteSecurityGroup",
                "ec2:AuthorizeSecurityGroupIngress",
                "ec2:AuthorizeSecurityGroupEgress",
                "ec2:RevokeSecurityGroupIngress",
                "ec2:RevokeSecurityGroupEgress",
                "ec2:RunInstances",
                "ec2:TerminateInstances",
                "ec2:StopInstances",
                "ec2:StartInstances",
                "ec2:CreateTags",
                "ec2:AllocateAddress",
                "ec2:AssociateAddress",
                "ec2:ReleaseAddress",
                "ec2:CreateKeyPair",
                "ec2:DeleteKeyPair"
            ],
            "Resource": "*"
        }
    ]
}
```

   - 오른쪽 아래 **[다음]**을 클릭합니다.
   - 정책 이름에 `Mission-EC2-VPC-Policy` 라고 적습니다.
   - 맨 아래 **[정책 생성]**을 클릭합니다.

3. **사용자 만들기**
   - 다시 왼쪽 메뉴에서 **[사용자(Users)]**를 클릭합니다.
   - 오른쪽 화면 상단의 주황색 **[사용자 생성]**을 클릭합니다.
   - **사용자 이름**: `mission-user` 라고 적습니다.
   - **[AWS Management Console에 대한 사용자 액세스 제공]** 체크박스를 **체크**합니다.
   - **'IAM 사용자를 생성하고 싶음'**을 선택합니다.
   - **콘솔 암호**: '사용자 지정 암호'를 선택하고, 로그인할 때 쓸 비밀번호를 입력합니다.
   - **'사용자는 다음 로그인 시 새 암호를 생성해야 합니다.'** 체크박스를 **해제**합니다. (번거로움을 줄이기 위함)
   - **[다음]** 클릭.
   - 권한 옵션에서 **[직접 정책 연결]**을 선택합니다.
   - 아래 검색창에서 방금 만든 `Mission-EC2-VPC-Policy`를 검색해서 **체크**합니다.
   - **[다음]** 클릭 -> **[사용자 생성]** 클릭.

4. **로그인 정보 저장 및 루트 계정 로그아웃**
   - 화면에 **'콘솔 로그인 URL'**이 나옵니다. (예: `https://123456789012.signin.aws.amazon.com/console`) 이 주소를 복사해서 메모장에 꼭 적어두세요!
   - 화면 오른쪽 맨 위의 본인 이름(계정명)을 클릭하고 **[로그아웃]**을 누릅니다.

## 🔑 2단계: 새로 만든 부캐(mission-user)로 접속하기

1. 방금 메모장에 적어둔 **콘솔 로그인 URL**로 접속합니다.
2. IAM 사용자 이름에 `mission-user`를 적고, 설정한 비밀번호로 로그인합니다.
3. 이제 여러분은 '루트 계정'이 아니라 안전하게 격리된 'IAM 사용자'로 접속하셨습니다!

## 🌍 3단계: 리전(Region) 설정하기 (아주 중요!)

AWS는 전 세계에 컴퓨터가 있습니다. 우리는 한국에 있으니 가장 빠른 '서울'에 서버를 만들어야 합니다.

1. 화면 오른쪽 맨 위, 종 모양 아이콘(알림) 옆에 있는 지역 이름을 클릭합니다. (보통 '미국 동부(버지니아 북부)' 등으로 되어 있을 수 있습니다.)
2. 목록에서 **[아시아 태평양 (서울) ap-northeast-2]**를 찾아 클릭합니다.
   *(주의: 이후 모든 실습은 화면 우상단이 '서울'인지 확인하고 진행하세요!)*

## 🏗️ 4단계: 나만의 격리된 네트워크 공간(VPC) 직접 만들기

네트워크를 직접 하나하나 조립해보는 과정입니다. 건물을 짓기 위해 땅을 사고(VPC), 구역을 나누고(Subnet), 대문을 달고(IGW), 길을 뚫어주는(Route Table) 과정입니다.

1. 화면 상단 검색창에 `VPC`를 검색해서 **[VPC]** 서비스로 들어갑니다.
2. 왼쪽 메뉴에서 **[VPC]** 하위의 **[내 VPC]** (또는 Your VPCs)를 클릭하고, 오른쪽 위 주황색 **[VPC 생성]**을 누릅니다.
   - **생성할 리소스**: `VPC만` 선택
   - **이름 태그**: `mission-vpc`
   - **IPv4 CIDR 블록**: `수동 입력` 선택 후, `10.0.0.0/16` 입력
   - 아래로 스크롤하여 **[VPC 생성]** 클릭. (땅을 샀습니다!)

3. 왼쪽 메뉴에서 **[서브넷]**을 클릭하고 **[서브넷 생성]**을 누릅니다.
   - **VPC ID**: 방금 만든 `mission-vpc` 선택
   - **서브넷 설정 1의 서브넷 이름**: `mission-public-subnet`
   - **가용 영역**: `아시아 태평양(서울) / ap-northeast-2a` 선택
   - **IPv4 CIDR 블록**: `10.0.1.0/24` 입력
   - **[서브넷 생성]** 클릭. (땅을 쪼개서 구역을 만들었습니다!)

4. 이 서브넷 안의 서버가 인터넷과 통신하려면, 서브넷 설정 중 하나를 바꿔야 합니다.
   - 방금 만든 `mission-public-subnet`을 목록에서 **클릭(체크)**합니다.
   - 오른쪽 위 **[작업]** -> **[서브넷 설정 편집]** 클릭.
   - **[퍼블릭 IPv4 주소 자동 할당 활성화]** 체크박스를 체크하고 맨 아래 **[저장]** 클릭.

5. 왼쪽 메뉴에서 **[인터넷 게이트웨이]**를 클릭하고 **[인터넷 게이트웨이 생성]**을 누릅니다.
   - **이름 태그**: `mission-igw`
   - **[인터넷 게이트웨이 생성]** 클릭.
   - 화면 위에 초록색 알림창이 뜨면 우측의 **[VPC에 연결]**을 클릭합니다.
   - **사용 가능한 VPC**: `mission-vpc` 선택 후 **[인터넷 게이트웨이 연결]** 클릭. (대문을 달았습니다!)

6. 길(라우팅)을 뚫어줄 차례입니다. 왼쪽 메뉴에서 **[라우팅 테이블]**을 클릭하고 **[라우팅 테이블 생성]**을 누릅니다.
   - **이름**: `mission-public-rt`
   - **VPC**: `mission-vpc` 선택
   - **[라우팅 테이블 생성]** 클릭.

7. 방금 만든 라우팅 테이블(mission-public-rt)의 상세 화면이 나옵니다.
   - 화면 중간쯤에 있는 **[라우팅]** 탭을 누르고 **[라우팅 편집]**을 클릭합니다.
   - **[라우팅 추가]** 클릭.
   - **대상(Destination)**: `0.0.0.0/0` 입력 (모든 외부 인터넷을 뜻함)
   - **대상(Target)**: 드롭다운을 눌러 `인터넷 게이트웨이`를 선택하고, 아까 만든 `mission-igw`를 클릭합니다.
   - **[변경 사항 저장]** 클릭. (인터넷으로 나가는 길을 만들었습니다!)

8. 마지막으로 이 길을 서브넷과 연결해줍니다.
   - 다시 라우팅 테이블(mission-public-rt) 화면 중간의 **[서브넷 연결]** 탭을 클릭합니다.
   - **[명시적 서브넷 연결 편집]** 클릭.
   - 아까 만든 `mission-public-subnet`의 체크박스를 체크하고 **[연결 저장]**을 누릅니다.

*(네트워크 구성이 완벽하게 끝났습니다! 정말 고생하셨습니다.)*

## 🛡️ 5단계: 방화벽(보안 그룹) 설정하기

보안 그룹은 서버의 앞문에서 허락된 요청만 들여보내는 경비원 역할을 합니다.

1. 화면 상단 검색창에 `EC2`를 검색해서 **[EC2]** 서비스로 들어갑니다.
2. 왼쪽 메뉴를 쭉 내려서 '네트워크 및 보안' 아래에 있는 **[보안 그룹]**을 클릭합니다.
3. 오른쪽 위 주황색 **[보안 그룹 생성]**을 누릅니다.
   - **보안 그룹 이름**: `mission-web-sg`
   - **설명**: `Web and SSH SG`
   - **VPC**: `mission-vpc` (반드시 기본 VPC의 x를 눌러 지우고, 방금 만든 mission-vpc를 선택해야 합니다!)
   
4. **인바운드 규칙** (외부에서 내 서버로 들어오는 허락 리스트)
   - **[규칙 추가]** 클릭
   - 유형: `HTTP` 선택 -> 소스: `Anywhere-IPv4` (0.0.0.0/0) 선택 (누구나 웹사이트는 봐야 하니까요)
   - 다시 **[규칙 추가]** 클릭
   - 유형: `SSH` 선택 -> 소스: `내 IP` (My IP) 선택 (서버 조종은 내 컴퓨터에서만 해야 하니까요. 자동으로 내 IP가 입력됩니다.)

5. 맨 아래 **[보안 그룹 생성]** 클릭.

## 💻 6단계: 우분투(Ubuntu) 가상 서버(EC2) 만들기

이제 진짜 컴퓨터 한 대를 빌려봅시다.

1. EC2 화면 왼쪽 메뉴에서 맨 위의 **[인스턴스]**를 클릭하고, 오른쪽 위 **[인스턴스 시작]**을 클릭합니다.
2. **이름 및 태그**: `mission-web-server` 라고 적습니다.
3. **애플리케이션 및 OS 이미지(Amazon Machine Image)**:
   - `Ubuntu` 아이콘을 클릭합니다.
   - 밑에 `Ubuntu Server 24.04 LTS (HVM), SSD Volume Type`이 선택되어 있는지 확인합니다. 밑에 **"프리 티어 사용 가능"** 마크가 있는지 꼭 확인하세요!
4. **인스턴스 유형**: `t2.micro` (프리 티어 사용 가능) 그대로 둡니다. (지역에 따라 t3.micro만 프리티어인 경우도 있습니다. 프리티어 글자가 있는 것을 선택하세요.)
5. **키 페어(로그인)**:
   - **[새 키 페어 생성]** 클릭.
   - 키 페어 이름: `mission-key`
   - 키 페어 유형: `RSA`
   - 프라이빗 키 파일 형식: `.pem` (윈도우 최신버전이나 맥은 pem이 편합니다.)
   - **[키 페어 생성]** 클릭.
   - **(주의!) `mission-key.pem` 파일이 컴퓨터(다운로드 폴더 등)에 다운로드됩니다. 이 파일은 절대 잃어버리거나 남에게 주면 안 됩니다. 이 열쇠가 없으면 평생 서버에 못 들어갑니다.**
6. **네트워크 설정**:
   - 우측에 있는 **[편집]** 버튼을 누릅니다.
   - VPC: `mission-vpc` 선택
   - 서브넷: `mission-public-subnet` 선택
   - 퍼블릭 IP 자동 할당: `활성화`로 되어있는지 확인.
   - 방화벽(보안 그룹): **'기존 보안 그룹 선택'**을 클릭하고, 아까 만든 `mission-web-sg`를 체크합니다.
7. **스토리지 구성**: `8 GiB`, `gp3` (기본값 유지)
8. 우측 하단의 **[인스턴스 시작]** 클릭!
9. 성공했다는 창이 뜨면, 인스턴스 ID(i-0abcdef...)를 클릭하거나 왼쪽 메뉴의 [인스턴스]로 가서 서버가 `실행 중(Running)` 상태가 될 때까지 1~2분 정도 기다립니다.

## 🔌 7단계: 내 맥북에서 EC2 서버에 접속하기 (SSH)

1. AWS 인스턴스 화면에서 방금 만든 `mission-web-server`를 클릭합니다.
2. 하단 세부 정보 창에서 **퍼블릭 IPv4 주소** (예: 13.125.xx.xx)를 복사해둡니다.
3. 맥북에서 **[터미널(Terminal)]** 앱을 엽니다. (Cmd + Space 누르고 terminal 검색)
4. 아까 다운받은 키 파일(`mission-key.pem`)이 있는 폴더로 이동합니다. 보통 다운로드 폴더에 있습니다.
   ```bash
   cd ~/Downloads
   ```
5. 열쇠 파일의 권한을 나만 읽을 수 있게 바꿉니다. (보안이 너무 열려있으면 접속을 거부합니다.)
   ```bash
   chmod 400 mission-key.pem
   ```
6. 드디어 서버에 원격 접속합니다! (아래 명령어의 IP 주소 자리에 복사해둔 IP를 넣으세요)
   ```bash
   ssh -i mission-key.pem ubuntu@여기에_퍼블릭_IP_주소를_적으세요
   ```
   - `Are you sure you want to continue connecting (yes/no)?` 라고 물어보면 `yes` 치고 엔터.
   - 초록색 글씨로 `ubuntu@ip-10-0-1-xx:~$` 처럼 나오면 접속 성공입니다! 내 맥북에서 멀리 서울에 있는 컴퓨터 안으로 들어온 것입니다.

## 🐳 8단계: 보너스 미션 - Docker로 Nginx 웹 서버 띄우기

이제 텅 빈 서버에 도커(Docker)를 설치하고 웹 서버를 띄워봅시다. 방금 띄운 터미널 창(우분투 내부)에 아래 명령어들을 한 줄씩 복사해서 엔터를 쳐주세요.

1. **소프트웨어 목록 업데이트**
   ```bash
   sudo apt-get update
   ```
2. **Docker 설치하기** (중간에 Y/n 물어보면 Y 누르고 엔터)
   ```bash
   sudo apt-get install -y docker.io
   ```
3. **Docker로 Nginx 웹 서버 컨테이너 실행하기**
   ```bash
   sudo docker run -d -p 80:80 nginx
   ```
   - 복잡한 문자열이 출력되면 도커 컨테이너가 성공적으로 실행된 것입니다.
4. **서버 내부에서 접속 잘 되는지 스스로 찔러보기**
   ```bash
   curl http://localhost
   ```
   - 화면에 `<!DOCTYPE html> ... Welcome to nginx! ...` 등등 HTML 코드가 쭈루룩 나오면 완벽하게 성공한 것입니다!

## 🎉 9단계: 외부 접속 확인하기 (미션 제출용)

1. 맥북에 있는 크롬이나 사파리 브라우저를 엽니다.
2. 주소창에 아까 복사했던 EC2 서버의 **퍼블릭 IPv4 주소**를 그대로 입력하고 엔터를 칩니다.
   - 예: `http://13.125.xx.xx`
3. 브라우저 화면에 **"Welcome to nginx!"** 라는 글자가 보인다면?
   - **축하합니다! 직접 만든 VPC와 방화벽을 뚫고 웹 서버 접속에 성공하셨습니다!**
   - 이 화면을 스크린샷 찍어서 미션 제출용으로 사용하시면 됩니다.

## 🧹 10단계: 실습 종료 후 과금 방지를 위한 리소스 완벽 정리

클라우드는 "쓴 만큼 돈을 낸다"가 원칙입니다. 켜두면 다음 달에 돈이 나갈 수 있으니, 실습이 끝나면 건물을 부수고 땅을 반납해야 합니다.
역순으로 지워야 오류가 나지 않습니다. (EC2 -> 보안 그룹 -> 라우팅 테이블/IGW/서브넷/VPC 순서)

1. **EC2 인스턴스 종료(삭제)**
   - AWS 콘솔의 **[EC2] -> [인스턴스]**로 갑니다.
   - `mission-web-server`를 체크합니다.
   - 오른쪽 위 **[인스턴스 상태] -> [인스턴스 종료(Terminate)]**를 클릭합니다. (중지가 아니라 종료를 누르셔야 완전 삭제됩니다.)
   - 상태가 `종료됨(Terminated)`으로 바뀔 때까지 기다립니다.

2. **보안 그룹 삭제**
   - 왼쪽 메뉴 **[보안 그룹]** 클릭.
   - `mission-web-sg` 체크 -> 우측 상단 **[작업] -> [보안 그룹 삭제]** 클릭.

3. **VPC 한 번에 삭제 (관련된 서브넷, IGW, 라우팅 테이블이 알아서 지워집니다)**
   - AWS 화면 위 검색창에 `VPC`를 쳐서 VPC 서비스로 이동합니다.
   - 왼쪽 메뉴 **[내 VPC]** 클릭.
   - `mission-vpc` 체크 -> 우측 상단 **[작업] -> [VPC 삭제]** 클릭.
   - 확인 창에 "delete" 라고 입력하고 삭제 버튼을 누릅니다.

이것으로 모든 리소스가 깔끔하게 삭제되었습니다. 체크리스트 파일(`docs/cleanup-checklist.md`)에 완료 표시를 하시면 됩니다! 정말 수고 많으셨습니다. 😊
