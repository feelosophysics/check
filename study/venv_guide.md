# 파이썬 가상환경(venv) 완벽 가이드 🚀

이 가이드는 파이썬 가상환경을 처음 접하는 입문자를 위해 작성되었습니다. 현재 교육용 PC 환경에서 겪고 있는 라이브러리 설치 문제(`externally-managed-environment`)를 가상환경을 사용하여 어떻게 완벽하게 해결할 수 있는지 단계별로 설명합니다.

---

## 1. 가상환경(Virtual Environment)이란 무엇인가요?

가상환경은 쉽게 말해 **"내 프로젝트 전용의 독립된 작은 방"**을 만드는 것입니다.

* **가상환경을 쓰지 않을 때 (기본 상태):** 컴퓨터 전체(시스템)에 라이브러리를 설치합니다. 이 경우 다른 프로젝트들과 버전 충돌이 나거나, 보안 정책에 의해 설치가 막힐 수 있습니다 (현재 겪으신 `externally-managed-environment` 에러가 바로 이것입니다).
* **가상환경을 쓸 때:** 내 프로젝트 폴더 안에 `.venv`라는 작은 폴더를 하나 만들고, 그 안에 필요한 라이브러리를 가둡니다. 시스템 영역을 전혀 건드리지 않기 때문에 **권한 에러 없이 자유롭게 패키지를 설치**할 수 있습니다.

---

## 2. [현재 상황] 가상환경 구축 Step-by-Step (macOS 기준)

프로젝트 폴더인 `check` 디렉토리 터미널에서 아래 단계를 순서대로 진행해 주세요.

### 1단계: 가상환경 방 만들기 (최초 1회만 실행)
터미널에서 아래 명령어를 입력하여 `.venv`라는 이름의 가상환경 폴더를 생성합니다.
```bash
python3 -m venv .venv
```
* 실행하고 나면 프로젝트 폴더 안에 `.venv`라는 폴더가 새로 생긴 것을 볼 수 있습니다.

### 2단계: 가상환경 안으로 들어가기 (활성화)
생성한 가상환경 방으로 입장합니다.
```bash
source .venv/bin/activate
```
* **성공 확인법:** 이 명령을 실행하고 나면 터미널 프롬프트 앞에 **`(.venv)`** 라는 글자가 생깁니다.
  * 예시: `(.venv) f22losophysics1091@c5r8s6 check %`
  * 이제부터 설치하는 모든 라이브러리는 시스템이 아닌 이 가상환경 내부에만 설치됩니다!

### 3단계: 필요한 라이브러리 설치하기
이제 권한 에러 없이 마음껏 라이브러리를 설치할 수 있습니다. 경고가 떴던 `pyyaml`과 필요했던 `python-dotenv`를 함께 설치합니다.
```bash
pip install pyyaml python-dotenv
```
* 더 이상 `externally-managed-environment` 에러가 나지 않고 깔끔하게 설치가 완료됩니다.

### 4단계: 프로그램 실행하기
설치가 완료되었으니 안심하고 프로그램을 실행합니다. (이때 `.env` 파일에 API 키가 적혀 있어야 합니다.)
```bash
python main.py commit
```
* 이제 설정 파일(`.ai-gitgen.yml`) 경고 문구 없이 아주 정상적으로 AI가 커밋 메시지를 생성합니다!

### 5단계: 가상환경 밖으로 나오기 (비활성화)
작업이 모두 끝나서 원래 컴퓨터 환경으로 돌아가고 싶다면 아래 명령어를 입력합니다.
```bash
deactivate
```
* 터미널 앞의 `(.venv)` 표시가 사라지며 가상환경에서 빠져나옵니다.

---

## 3. 앞으로 작업할 때 기억해야 할 핵심 루틴 💡

내일 컴퓨터를 켜고 다시 이 작업을 하려고 할 때, 매번 1단계(생성)부터 할 필요는 없습니다. 아래 루틴만 기억하세요!

1. **터미널을 켠다.**
2. **가상환경을 활성화한다:** `source .venv/bin/activate` (터미널 앞에 `(.venv)` 확인)
3. **코딩 및 명령어 실행을 한다:** `python main.py commit`
4. **끝나면 종료한다:** `deactivate`

---

## 4. ⚠️ 주의할 점 (Git 관리 관련)

가상환경을 만들면 내부에 패키지들이 다운로드되어 `.venv` 폴더의 용량이 매우 커집니다. 이 폴더는 굳이 GitHub에 올릴 필요가 없으므로 **`.gitignore` 파일에 반드시 `.venv`가 포함되어 있어야 합니다.**

현재 프로젝트의 `.gitignore` 파일에는 이미 `.venv` 혹은 `venv`가 등록되어 있어 Git이 알아서 제외해 주므로 안심하셔도 됩니다!
